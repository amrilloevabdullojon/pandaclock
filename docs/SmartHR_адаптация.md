**Версия:** 1.0

---

## 1. Что такое SmartHR

**SmartHR** — это очень большой коммерческий продукт от компании Dreamguys / Dreamstechnologies:

- **Figma-версия** — бесплатная (наш выбор)
- **Демо:** https://smarthr.dreamstechnologies.com/
- **Платные имплементации:** React + Next.js, Angular, Vue, HTML, Laravel ($59-79 на ThemeForest)
- **12 дашбордов:** Admin, Employee, Leads, Deals, HR, Payroll, Finance, Recruitment, Attendance, Asset, IT Admin, Help Desk

**Это плюс и минус одновременно:**
- ➕ Покрывает 80-90% всего нашего roadmap, а не только MVP
- ➕ Есть готовый React + Next.js код (можно купить)
- ➖ Очень "корпоративный" и слегка устаревший стиль
- ➖ Много модулей, которые нам не нужны (CRM, IT Admin, Help Desk, инвойсы)

### ❌ Игнорируем (есть в SmartHR, нам не нужно в MVP)

- CRM (Contacts, Companies, Deals, Leads, Pipeline) — не наш фокус
- Payroll и финансовые модули — Этап 4
- Recruitment (Job listings, Candidates) — не в MVP
- Invoices, Expenses, Assets — не в HR
- IT Admin, Help Desk — отдельный продукт
- Training & Development — Этап 3+
- Video / Voice calls (через LiveKit будем отдельно) — Этап 2
- To-do lists, File Manager — есть в задачах
- Promotion management — Этап 3+

**Итого:** оставляем ~40% функций SmartHR, игнорируем ~50%, добавляем своё ~10%.

## 3. Стратегическое решение: только Figma или купить код?

### Вариант 1: Только Figma (бесплатно)
- Скачиваем бесплатный SmartHR Figma kit
- Используем как визуальный референс
- Код пишем сами на shadcn/ui

**Плюсы:** уникальный код, контроль, $0
**Минусы:** + 4-6 недель работы фронтенд-разработчика

### Вариант 2: Купить React + Next.js версию ($59-79) ⭐ РЕКОМЕНДУЮ
- Покупаем [SmartHR React + Next.js Admin Template](https://themeforest.net/item/smarthr-react-admin-template/28253842)
- Берём готовые React-компоненты
- Адаптируем под наш бэкенд
- Заменяем дизайн на свой бренд

**Плюсы:** экономия 4-6 недель работы = $4,000-8,000 экономии при цене $79
**Минусы:**
- Используется Bootstrap 5, а наш стек — Tailwind + shadcn/ui → нужно либо мигрировать, либо адаптировать
- Код может быть не идеального качества (типичная проблема ThemeForest)
- Зависимость от сторонней библиотеки

### Вариант 3: Гибрид (мой реальный совет)
- **Figma:** SmartHR как референс
- **Код:** пишем на нашем стеке (Tailwind + shadcn/ui), вдохновляясь структурой SmartHR
- **НЕ покупаем** платную версию (Bootstrap не совместим с нашим стеком)

**Плюсы:** наш стек чистый, нет тех.долга от Bootstrap, дизайн под наш бренд
**Минусы:** немного больше работы чем при готовом коде

## 4. План действий (по шагам)

### Шаг 1: Скачать Figma (15 минут)
1. Зайти в [SmartHR Figma Community](https://www.figma.com/community/file/1539212751721531783/smarthr-free-figma-ui-kit-for-hrm-employee-management)
2. Кнопка "Open in Figma" → "Duplicate" в свой аккаунт
3. Теперь у вас личная копия — можно редактировать

### Шаг 2: Изучить демо (1-2 часа)
- Открыть https://smarthr.dreamstechnologies.com/
- Пройти по всем дашбордам и модулям
- Сделать скриншоты тех экранов, которые понравились
- Отметить что переделать (UX, перегруженность, цвета)

### Шаг 3: Создать дизайн-систему (1 неделя)
Зафиксировать в Figma:
- **Цвета:** наша палитра вместо SmartHR (рекомендую #4F46E5 + #10B981)
- **Шрифт:** Inter (бесплатный, чистый) или Manrope
- **Логотип:** наш
- **Spacing scale:** 4/8/16/24/32/48
- **Радиусы:** 6px (карточки), 8px (модалки), 4px (кнопки)

### Шаг 4: Адаптировать ключевые экраны (2-3 недели)
Приоритет:
1. Landing page (новый, на основе B2Base)
2. Регистрация компании (новый, multi-step)
3. Dashboard сотрудника (адаптация Employee Dashboard)
4. Учёт времени (адаптация Attendance Employee)
5. Список сотрудников (адаптация Employee List)
6. Карточка сотрудника (адаптация Employee Profile)
7. Задачи + канбан (адаптация Tasks + Kanban)
8. Дашборд руководителя (адаптация HR Dashboard, упрощённо)

### Шаг 5: Mobile-экраны (1-2 недели)
SmartHR имеет Ionic-версию для мобильного. Но мы на React Native — берём только идеи UX.

Ключевые mobile-экраны:
- Логин / OTP
- Главная (большая кнопка "Начать день")
- Список задач
- Карточка задачи
- Заявки
- Чат
- Профиль

## 5. Что улучшить в дизайне SmartHR

SmartHR — хороший стартовый набор, но имеет **типичные проблемы корпоративных шаблонов**. Рекомендую при адаптации:

### Дашборды
- ❌ SmartHR: 10+ виджетов на главной → перегружено
- ✅ Наш подход: 4-6 ключевых KPI, остальное по клику

### Цветовая палитра
- ❌ SmartHR: жёлто-оранжевая, выглядит "1С-подобно"
- ✅ Наш подход: фиолетовый primary (#4F46E5) + зелёный success (#10B981) + красный warning (#EF4444)

### Типографика
- ❌ SmartHR: мелкие шрифты, плотные таблицы
- ✅ Наш подход: больше воздуха, читаемость > плотность

### Иконки
- ❌ SmartHR: FontAwesome (устаревшие)
- ✅ Наш подход: Lucide Icons (современные, идут с shadcn/ui)

### Mobile
- ❌ SmartHR Ionic версия выглядит web-портом
- ✅ Наш подход: native-friendly UX на React Native + NativeWind
