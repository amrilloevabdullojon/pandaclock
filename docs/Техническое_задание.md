# Техническое задание
## HR/CRM система управления персоналом (SaaS)

**Версия:** 2.1
**Дата:** 2026-05-24

### Ключевые параметры проекта
- **Модель:** SaaS (multi-tenant) — несколько компаний-клиентов на одной платформе
- **Целевой размер компании-клиента:** от 3000 сотрудников
- **Основной рынок:** Узбекистан
- **Перспектива:** расширение на другие страны СНГ и далее

---

## 1. Пользовательские роли

### 1.1 Сотрудник
- Отмечает начало и окончание рабочего дня
- Отмечает перерывы (обед, кофе-брейк)
- Просматривает свой график работы
- Видит задачи на день/неделю/месяц
- Отмечает статусы задач (готово / в процессе / отклонено)
- Участвует в общем чате отдела
- Проводит и присоединяется к созвонам
- Подаёт заявки на отпуск / больничный
- Получает уведомления (push, email)
- Заполняет ежедневный чек-ин по самочувствию (опционально)

### 1.2 Руководитель отдела
- Видит всех сотрудников своего отдела (ФИО, должность, статус)
- Контролирует посещаемость (онлайн-статус, опоздания)
- Создаёт и назначает задачи
- Утверждает / отклоняет заявки на отпуск
- Просматривает отчёты по отделу
- Видит KPI и статистику по сотрудникам
- Участвует в чатах и созвонах

### 1.3 HR-администратор
- Управляет карточками сотрудников (приём, увольнение, перевод)
- Загружает и хранит документы сотрудников
- Контролирует процесс онбординга новых сотрудников
- Формирует и экспортирует HR-отчёты
- Управляет отпускным графиком
- Отслеживает сроки документов (паспорт, медкнижка, сертификаты)

### 1.4 Администратор компании (CEO / владелец компании-клиента)
- Создаёт отделы и подразделения внутри своей компании
- Назначает руководителей
- Управляет ролями и правами доступа в рамках своей компании
- Просматривает сводные отчёты по всей компании
- Настраивает параметры компании (рабочее время, праздники, политики)
- Управляет подпиской и оплатой

### 1.5 Супер-администратор платформы (наша команда)
- Управляет компаниями-клиентами (регистрация, отключение)
- Контролирует тарифные планы и биллинг
- Мониторинг состояния платформы
- Поддержка клиентов
- Управляет глобальными настройками платформы
- **Не имеет доступа** к персональным данным сотрудников клиентов (изоляция данных)

## 2. Функциональность роли «Сотрудник»

### 2.1 Учёт рабочего времени
- Кнопка **«Начать рабочий день»** с фиксацией времени и геолокации/IP
- Кнопка **«Завершить рабочий день»**
- Учёт перерывов (старт/стоп обед)
- Автоматическая фиксация времени в системе
- Просмотр истории своего рабочего времени за период
- Offline-режим (на мобильном) — отметка сохраняется и синхронизируется при появлении интернета

### 2.2 Задачи
- Просмотр задач на день / неделю / месяц
- Фильтрация по статусу и приоритету
- Изменение статуса: **Новая → В процессе → Готово / Отклонено**
- Комментарии к задаче
- Прикрепление файлов
- Учёт по статусам за день / неделю / месяц
- Уведомление о новой задаче

### 2.3 График работы
- Просмотр графика на неделю / месяц
- Интеграция с календарём (Google Calendar, Outlook)
- Отображение запланированных встреч и созвонов

### 2.4 Коммуникации
- Общий чат отдела
- Личные сообщения коллегам
- Видео-созвоны (1-на-1 и групповые)
- Демонстрация экрана в созвонах
- История чатов с поиском

### 2.5 Заявки
- Заявка на отпуск (с выбором дат и типа)
- Заявка на больничный
- Заявка на отгул
- Просмотр статуса своих заявок

### 2.6 Уведомления
- Push-уведомления (мобильное приложение)
- Email-уведомления (резервный канал)
- Напоминание о начале рабочего дня
- Уведомления о новых задачах и дедлайнах
- Уведомления о сообщениях в чате
- Уведомления о решении по заявкам

### 3.3 Заявки

- Утверждение / отклонение с комментарием
- История решений

### 3.4 Отчёты по отделу

- Опоздания и ранние уходы
- Переработки
- Процент выполнения задач (KPI)

- Экспорт в **Excel / PDF**

### 3.5 KPI и метрики
- Процент выполненных задач в срок
- Среднее время выполнения задачи
- Активность сотрудника (часы, задачи)

## 6. Нефункциональные требования

### 6.1 Безопасность
- Авторизация по логину/паролю
- Двухфакторная аутентификация (2FA)
- Восстановление пароля по email
- Хранение паролей в виде хеша (bcrypt / argon2)
- Шифрование данных при передаче (HTTPS / TLS 1.3)
- Шифрование данных в базе данных (at rest)
- Журнал аудита действий (audit log)
- Защита от типовых атак (OWASP Top 10)
- Регулярное резервное копирование (ежедневно)

### 6.2 Защита персональных данных

**Применимое законодательство:**
- **Узбекистан** — Закон «О персональных данных» №547 от 02.07.2019
- **Россия** (если будут клиенты) — 152-ФЗ
- **EU** (при международном расширении) — GDPR

**Ключевое требование Узбекистана:** персональные данные граждан Узбекистана должны храниться на серверах, физически расположенных в Узбекистане (локализация данных).

**Требования к системе:**
- Согласие сотрудника на обработку персональных данных при регистрации
- Право на доступ — сотрудник может скачать все свои данные
- Право на удаление — по запросу сотрудника (с учётом юридических ограничений)
- Политика хранения данных: рабочие записи — 3 года, документы — 5 лет
- Уведомление о нарушениях безопасности в течение 72 часов
- Экспорт персональных данных в стандартном формате (JSON / CSV)
- **Регистрация в реестре операторов персональных данных** Узбекистана (организационное)
- Возможность размещения данных в разных регионах в зависимости от страны клиента (data residency)
- Договор обработки данных (DPA) с каждой компанией-клиентом

### 6.3 Производительность и масштабируемость
- Время отклика API ≤ 300 мс (95-й перцентиль)
- Загрузка страницы веб-приложения ≤ 2 секунды
- Поддержка до **5000 одновременных пользователей на одну компанию-клиента**
- Расчётная нагрузка: 50+ компаний-клиентов × 3000+ сотрудников = **150 000+ активных пользователей**
- Доступность системы ≥ 99.9% (SLA для SaaS)
- Горизонтальное масштабирование (микросервисы / kubernetes)
- CDN для статики (особенно для мобильных приложений)
- Кэширование (Redis) для тяжёлых запросов
- Очереди задач (RabbitMQ / Kafka) для асинхронной обработки (отчёты, экспорт, уведомления)

### 6.4 Платформы
| Платформа | Назначение | Технология (рекомендация) |
|-----------|------------|---------------------------|
| **Веб-приложение** | Основное для руководителей, HR, админов | React / Next.js |
| **Mobile iOS** | Для сотрудников (отметка времени, чат) | React Native / Flutter |
| **Mobile Android** | Для сотрудников | React Native / Flutter |

### 6.5 Интеграции
- Google Calendar / Outlook (синхронизация графика)
- Email (SMTP) для уведомлений
- Push-уведомления (FCM для Android, APNs для iOS)
- Опционально: Telegram-бот для уведомлений

### 6.6 Локализация (i18n)
- **Узбекский (латиница)** — основной
- **Узбекский (кириллица)** — альтернативный
- **Русский** — основной
- **Английский** — для международного расширения
- Архитектура с поддержкой добавления новых языков без изменения кода
- Локализация дат, чисел, валют (UZS, RUB, USD)
- Часовые пояса (каждая компания и сотрудник могут иметь свой)

### 6.7 Multi-tenancy (мультиарендность) — SaaS архитектура

**Решение:** система строится как multi-tenant SaaS.

#### Подход к изоляции данных
- **Schema-per-tenant** (рекомендуется для 3000+ сотрудников на клиента) — каждая компания получает отдельную схему в БД. Хорошая изоляция, легче бэкапы, проще GDPR-удаление.
- Альтернатива: **shared schema с tenant_id** — проще в реализации, но требует тщательной фильтрации в каждом запросе.

#### Требования к multi-tenant архитектуре
- Полная изоляция данных между компаниями
- Регистрация новой компании (self-service или через менеджера)
- Поддомен или путь для каждой компании: `companyname.hrcrm.uz`
- Tenant-aware кэширование (Redis с префиксами)
- Tenant-aware фоновые задачи
- Метрики и логирование с разделением по tenant

#### Биллинг и тарифы
- Тарифные планы (Starter / Business / Enterprise)
- Расчёт по количеству активных сотрудников
- Интеграция с платёжными системами Узбекистана: **Click, Payme, Uzcard**
- Интеграция с международными платежами (Stripe) — для расширения
- Триал-период (14-30 дней)
- Автоматическое выставление счетов
- Уведомления об оплате

## 7. Этапы разработки (рекомендуемый план)

> **Важно:** для SaaS multi-tenancy закладывается в архитектуру с первого дня. Переделывать single-tenant в multi-tenant позже — это полная переписка.

### Этап 0: Архитектура и инфраструктура (1 месяц)
- Проектирование multi-tenant архитектуры (схема изоляции данных)
- Развёртывание MVP на **Vercel** (для быстрого старта)
- Настройка CI/CD, мониторинга, логирования
- Базовая схема БД с поддержкой tenant
- Авторизация и регистрация компаний

**Важно:** код должен быть портируемым — без жёсткой привязки к Vercel-специфичным фичам (или с абстракцией над ними), чтобы при миграции на узбекские серверы не пришлось переписывать существенную часть системы.

### Этап миграции (после набора первых клиентов)
- Подбор хостинг-провайдера в Узбекистане
- Миграция production-окружения на локальные серверы
- Соответствие требованию о хранении ПД на территории Узбекистана

### Этап 1: MVP (3-4 месяца)
- Регистрация компании-клиента (self-service)
- Авторизация сотрудников, роли (Сотрудник, Руководитель)
- Учёт рабочего времени (старт/стоп, перерывы)
- Базовые задачи (создание, назначение, статусы)
- Простые отчёты (часы, задачи)
- Mobile-приложение для сотрудников (iOS + Android)
- Web-приложение для руководителей
- Базовый биллинг (один тариф) с интеграцией Click / Payme

### Этап 2 (4-7 месяцев)
- Роль HR-администратора
- Документы сотрудников
- Заявки на отпуск / больничный
- Чаты отделов
- Push-уведомления (FCM, APNs)
- Экспорт отчётов (Excel / PDF)
- Несколько тарифных планов
- Триал-период

### Этап 3 (7-10 месяцев)
- Видео-созвоны
- Роль Администратора компании (расширенная)
- Онбординг новых сотрудников
- KPI и расширенная аналитика
- Интеграции с календарями
- 2FA, audit log
- Инструменты для соответствия закону Узбекистана о персональных данных
- Дашборд супер-администратора платформы

### Этап 4 (10-12 месяцев)
- Расширенная локализация (узбекский латиница/кириллица, английский)
- Интеграция с международными платежами (Stripe)
- Модуль расчёта зарплаты
- Telegram-бот для уведомлений
- API для клиентов (интеграция с 1С и другими системами)

### Этап 5 (опционально, после launch)
- Peer recognition / kudos
- Mood check-ins
- AI-аналитика (предсказание выгорания, рекомендации)
- Расширение на другие страны СНГ

## 8. Принятые решения

- ✅ **Multi-tenancy:** SaaS (multi-tenant)
- ✅ **Размер компании-клиента:** 3000+ сотрудников
- ✅ **Основной рынок:** Узбекистан, с расширением на другие страны
- ✅ **Хостинг:**
  - **Этап тестирования / MVP:** Vercel (быстрый старт, удобный деплой)
  - **Production:** переход на локальные серверы в Узбекистане (для соответствия закону о персональных данных)
- ✅ **Модель продаж:** гибридная — self-service регистрация + продажи через менеджеров (для крупных корпоративных клиентов)
- ✅ **Целевые отрасли (приоритет):**
  1. **IT / офис** — компании с гибким графиком, удалёнкой, фокус на задачах и KPI
  2. **Банки / финансы** — строгая безопасность, compliance, документооборот
  3. **Колл-центры** — высокая текучесть, сменный график, интеграции с телефонией
  4. **HoReCa (отели/рестораны)** — сменное планирование, мульти-локации, сезонность
- ✅ **Приоритет продаж:** IT-компании первыми, HoReCa вторыми, банки и колл-центры — после построения базы
- ✅ **Целевые показатели первого года:** 30-50 платящих компаний, ~$10k MRR к концу года
- ✅ **Тарифная сетка:**
  - **Starter** (бесплатно, до 10 сотрудников) — для лидогенерации
  - **Business** (50,000 UZS + 5,000/сотрудник, 11-100) — основной тариф
  - **Pro** (30,000 UZS/сотрудник, 100-500) — для средних компаний
  - **Enterprise** (договорная, от 1.5M UZS/мес, 500+) — корпоративный
- ✅ **Геолокация при отметке:** настраиваемая компанией (выкл / радиус от офиса / мульти-точки)
- ✅ **Видео-созвоны:** LiveKit (Cloud в начале → self-hosted)
- ✅ **Интеграция с 1С:** не в MVP, добавится в Этап 3-4
- ✅ **Биометрия:** только встроенная (Face ID / Touch ID / Fingerprint) для входа в приложение
- ✅ **Хостинг в Узбекистане (приоритет):** Uztelecom Cloud → EPAM/Cloud4U → on-premise
- ✅ **SSO:** Google + Microsoft в Этап 2
- ✅ **АТС для колл-центров:** Asterisk первым (70%+ рынка)
- ✅ **Записи звонков:** только метаданные в MVP, аудиофайлы — премиум-опция позже
- ✅ **Терминалы HoReCa:** да, в Этап 2 (веб-версия для планшета + PIN/QR)
- ✅ **Мульти-юрлица в HoReCa:** да, поддержка с Этапа 2
- ✅ **Учёт чаевых:** оба варианта (tronk + индивидуальный) в Этап 3
- ✅ **ЭЦП Узбекистана (E-IMZO):** в Этап 4 (банковский модуль)
- ✅ **White-label:** не в MVP, премиум-фича в Этап 3+
- ✅ **Compliance:** требования ЦБ РУз сразу для банков, ISO 27001 — после 100+ клиентов

## 9. Отраслевая специфика

> Каждая из 4 выбранных отраслей имеет уникальные требования. Эти фичи строятся **поверх базовой системы** и активируются модулем (включается на уровне компании-клиента в зависимости от её отрасли).

### 9.1 IT / Офисные компании

**Особенности:**
- Гибкий график, частая удалённая работа
- Сотрудники сами планируют рабочее время
- Главная метрика — задачи и результат, а не часы
- Высокий уровень технической грамотности пользователей

**Специфические фичи:**
- Гибкий учёт времени (можно начать день в любое время)
- Поддержка удалённой работы (без обязательной геолокации)
- Интеграции:
  - **Jira / YouTrack** — синхронизация задач
  - **GitHub / GitLab** — связь задач с PR/commits
  - **Slack** — уведомления и команды через бота
  - **Google Workspace / Microsoft 365** — календарь, авторизация (SSO)
  - **Zoom / Google Meet** — видео-созвоны
- KPI на основе закрытых задач, code reviews
- Sprint / Agile-friendly отчётность
- Async-коммуникация (чаты важнее созвонов)

### 9.2 Банки / Финансовые организации

**Особенности:**
- Строгое регулирование (ЦБ Узбекистана)
- Высокие требования к безопасности и compliance
- Жёсткий рабочий регламент
- Многоуровневая иерархия и согласования

**Специфические фичи:**
- **Обязательное** 2FA для всех сотрудников
- Расширенный audit log (фиксация ВСЕХ действий с timestamps)
- IP-whitelist (доступ только из офисной сети)
- Запрет/контроль удалённой работы (настраивается)
- Многоуровневое согласование заявок (например, отпуск: руководитель → начальник управления → HR)
- Compliance-отчёты для регулятора
- Учёт обучений и сертификаций (обязательные курсы)
- Конфиденциальные документы с разграничением доступа
- Электронная подпись документов
- Резервирование сессий (запись действий для расследований)

### 9.3 Колл-центры

**Особенности:**
- Огромный штат операторов (500-3000+ на одну компанию)
- Высокая текучесть кадров (до 50% в год)
- Сменный график 24/7
- Жёсткий контроль KPI по звонкам
- Реал-тайм мониторинг статусов

**Специфические фичи:**
- **Сменное планирование** (графики 5/2, 2/2, 12/24, ночные смены)
- Учёт перерывов с точностью до минуты (technical/lunch/personal)
- Реал-тайм дашборд статусов операторов:
  - 🟢 Доступен / 🔵 В разговоре / 🟡 Перерыв / 🔴 Не активен
- Интеграции с **АТС / телефонией**:
  - Asterisk (open source, популярен в Узбекистане)
  - Bitrix24 Telephony
  - Webitel
  - 3CX
- KPI операторов:
  - Количество звонков
  - Среднее время разговора (AHT)
  - Конверсия
  - First Call Resolution (FCR)
  - Качество звонков (после оценки супервайзером)
- **Быстрый онбординг** (упрощённая регистрация массовых новичков, импорт списка из Excel)
- Обучающий модуль (видео, тесты для новых операторов)
- Игрофикация (рейтинги, бейджи, бонусы)
- Запись и оценка звонков (если интеграция с АТС позволяет)

### 9.4 HoReCa (отели и рестораны)

**Особенности:**
- Сменный график, часто меняющийся
- Несколько локаций в сети (рестораны/отели)
- Разные позиции с разными ставками (повар, официант, бармен, администратор)
- Сезонность (летние террасы, новогодние смены)
- Учёт чаевых
- Часто молодой персонал, не всегда со смартфонами

**Специфические фичи:**
- **Smart scheduling** — авто-составление графика смен с учётом:
  - Доступности сотрудников
  - Прогноза загрузки (по дням недели, сезону)
  - Минимума/максимума часов
  - Стоимости часа
- **Мульти-локации** — сотрудник может работать в нескольких ресторанах сети
- Геолокация при отметке (по точке/ресторану)
- **Терминалы для отметки** (планшеты в служебке) — для сотрудников без смартфонов
- PIN-код или QR-код для отметки на терминале
- Учёт позиций с разными почасовыми ставками
- Учёт чаевых (общая касса / индивидуальные)
- Запросы на обмен сменами между сотрудниками
- Быстрая публикация вакансий (для покрытия пиков)
- Уведомление сотрудников о новых сменах за день/неделю
- Расчёт зарплаты с учётом смен, ставок, чаевых

### 9.5 Архитектурное решение: модульность

Учитывая что отрасли разные, система проектируется как **модульная**:

### 9.6 Приоритет разработки отраслевых модулей

**MVP (Этап 1):** базовая система, подходит для IT-компаний (минимум кастомизации)

**Этап 2:** модуль для HoReCa (smart scheduling, мульти-локации, терминалы) — большой рынок, понятная боль, легче продать

**Этап 3:** модуль для колл-центров (интеграция с АТС, реал-тайм дашборд) — высокий чек, но сложная разработка

**Этап 4:** модуль для банков (compliance, многоуровневое согласование, расширенный audit) — самый сложный, но самый дорогой клиент
