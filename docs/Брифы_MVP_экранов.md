# 1. Регистрация компании

## Назначение
Первый экран самостоятельной регистрации компании-клиента. Должен быть максимально простым — конверсия в активацию = главный KPI продукта.

## Принцип: 4 шага, не больше

Многошаговая форма (wizard) с прогресс-баром сверху. На каждом шаге — минимум полей.

## Шаги детально

### Шаг 1: О компании
- Название компании (text input)
- Поддомен (text input с автогенерацией из названия + валидация уникальности в реальном времени)
- Сфера (select)
- Размер компании (radio buttons как chips)

**Валидация:** все поля обязательны. Поддомен — только латиница, цифры, дефис.

### Шаг 2: Об администраторе

- Email строгая валидация (regex)

## Технические детали

- Состояние формы хранится в URL (`?step=2`) — можно вернуться

- Все запросы валидации — debounced (300ms)
- После шага 2: создание tenant в БД, отправка confirmation email
- После шага 4: автоматический login + редирект в dashboard

# 2. Login + 2FA

## Назначение
Простой и быстрый вход. 2FA опционально (включается в Этапе 2).

## Layout (Login)

```

│                                      │

│   │ Email                       │    │
│   │ [anna@example.com]          │    │
│   │                             │    │
│   │ Пароль                      │    │
│   │ [_________________] [👁]     │    │
│   │                             │    │
│   │ ☐ Запомнить меня            │    │
│   │            [Забыли пароль?] │    │
│   │                             │    │
```

## Layout (2FA подтверждение)

После успешного ввода логина/пароля, если 2FA включена:

```
┌──────────────────────────────────────┐
│        🐼 Pandaclock                  │
│                                      │
│   ┌────────────────────────────┐    │

│   │                             │    │
│   │  [_][_][_]  [_][_][_]       │    │ ← 6-цифровой input
│   │                             │    │

│   │                             │    │

│   └────────────────────────────┘    │
└──────────────────────────────────────┘
```

## Layout (Восстановление пароля)

```
┌──────────────────────────────────────┐
│   ┌────────────────────────────┐    │
│   │ Восстановление пароля       │    │
│   │                             │    │
│   │ Введите email, на который   │    │
│   │ зарегистрирован аккаунт.    │    │
│   │                             │    │
│   │ Email                       │    │
│   │ [____________________]      │    │
│   │                             │    │
│   │ [Отправить ссылку]          │    │
│   │                             │    │
│   │ [← Назад ко входу]          │    │
│   └────────────────────────────┘    │
└──────────────────────────────────────┘
```

После клика → экран с подтверждением "Если такой email зарегистрирован, на него отправлена ссылка."

## Технические детали

- Поле email: типа `email`, autocomplete `email`
- Поле пароль: autocomplete `current-password`
- При ошибке: единое сообщение "Неверный email или пароль" (не раскрываем что именно неверно)
- Rate limiting: 5 попыток за 5 минут → блокировка с CAPTCHA
- После 3-х неудач — показать ссылку "Восстановить пароль"
- 2FA: 6-цифровой OTP, поддержка Google Authenticator / Authy
- Резервные коды: 10 одноразовых при включении 2FA

# 3. Список сотрудников + карточка

## 3.1 Список сотрудников (Web)

### Назначение
Главный экран HR / руководителя — обзор всех сотрудников компании или отдела с возможностью быстрого поиска и фильтрации.

### Layout

```

│            ├────────────────────────────────────────────────────────┤

│            │  └──────────┴──────────┴──────────┴──────────┘       │
│            │                                                       │
│            │  [🔍 Поиск...]  [▼ Отдел] [▼ Должность] [▼ Статус]   │
│            │                                                       │
│            │  ┌────────────────────────────────────────────────┐  │
│            │  │ Сотрудник           Отдел       Должность     │  │
│            │  ├────────────────────────────────────────────────┤  │
│            │  │ [👤] Анна Каримова  Маркетинг   Менеджер  🟢   │  │

└────────────┴────────────────────────────────────────────────────────┘
```

### Особенности

**Поиск и фильтры:**
- Поиск по ФИО, email, телефону — debounced 300ms
- Фильтры: Отдел (multi), Должность (multi), Статус (active/inactive/on-leave/terminated)
- "Чистая" страница: показывается всё. Применённые фильтры — chips сверху таблицы с возможностью убрать

**Таблица:**
- Столбцы: аватар, ФИО + email, отдел, должность, статус, действия (⋮)
- Sortable: по ФИО, отделу, дате найма
- Клик на строку → переход в карточку сотрудника
- Контекстное меню (⋮): "Открыть карточку", "Написать в чат", "Назначить задачу", "Уволить"

**Bulk actions (если выбрано несколько):**
- Чекбокс слева в каждой строке
- При выборе появляется панель сверху: "Выбрано 3 [Назначить отдел] [Экспорт] [Удалить]"

**Кнопка "Добавить":**
- Dropdown: "+ Одного сотрудника" / "+ Импорт из Excel" / "+ Создать ссылку-приглашение"

### Mobile-версия
- Таблица превращается в карточки (1 в ряд)
- Фильтры — модальная панель снизу
- Поиск выезжает сверху

### Вкладки

**1. Профиль** (по умолчанию)
- Основная информация (ФИО, контакты, дата рождения)
- Работа (должность, отдел, график, дата найма)
- Кнопка "Редактировать"

**2. Время**
- Календарь с цветовой кодировкой (зелёный — пришёл, жёлтый — опоздал, серый — выходной)
- Статистика за месяц/квартал/год (часы, опоздания, переработки)
- Детальная таблица входов/выходов

**3. Задачи**
- Список задач, назначенных этому сотруднику
- Фильтры по статусу
- Метрики: % выполнения, среднее время

**4. Заявки**
- История всех заявок (отпуска, больничные)
- Статусы

**5. Документы** (для HR)
- Список документов (паспорт, договор, диплом)
- Сроки действия (предупреждение, если истекает)
- Кнопка "Добавить документ"

### Меню "⋮ Действия" (правый верхний угол)

- 💬 Написать в чат
- 📞 Позвонить
- ✅ Назначить задачу
- ✈️ Создать заявку от имени
- 🔄 Перевести в другой отдел
- 📤 Экспорт данных
- ❌ Уволить (с подтверждением)

- Сортируемая таблица

**Календарь:**
- Месячный/недельный вид
- Задачи отображаются как события на дате дедлайна
- Цвет = статус
- Drag по календарю → изменение дедлайна
