# Roadmap MVP
## HR/CRM SaaS — детальный план разработки

**Версия:** 1.0
**Дата:** 2026-05-24
**Длительность MVP:** 16 недель (4 месяца)
**Спринты:** 2 недели

---

## Цель MVP

Запустить работающий продукт, который:
1. Позволяет компании зарегистрироваться и пригласить сотрудников
2. Сотрудники отмечают время, видят и закрывают задачи
3. Руководители создают задачи, видят отчёты
4. Работает на web + iOS + Android
5. Принимает оплату через Click/Payme
6. Подходит для IT-компаний и небольшого HoReCa в первой версии

## Риски и митигация

| Риск | Вероятность | Митигация |
|------|-------------|-----------|
| Multi-tenant архитектура сделана неправильно | Высокая | Начать со schema-per-tenant сразу, не откладывать |
| Vercel не справится с нагрузкой | Средняя | Иметь готовый план миграции (Docker + VPS) |
| Push-уведомления нестабильно работают | Средняя | Использовать Expo (упрощает FCM/APN) |
| Click/Payme integration занимает дольше | Высокая | Резерв 1-2 спринта; временно — ручная оплата |
| Команда не успевает за 4 месяца | Высокая | Урезать scope; сначала только web, mobile потом |
| Тестовые клиенты не дают обратную связь | Средняя | Договориться с 3-5 клиентами ДО старта разработки |
| Закон о хранении ПД блокирует Vercel | Низкая (для теста) | Чёткая коммуникация: на Vercel — только демо/тест |

## Что должно быть готово к концу MVP

**Технически:**
- ✅ Регистрация компании, авторизация, multi-tenancy
- ✅ Управление сотрудниками и отделами
- ✅ Учёт рабочего времени (с геолокацией)
- ✅ Задачи (создание, статусы, комментарии)
- ✅ Заявки (отпуск, больничный)
- ✅ Базовые отчёты + экспорт в Excel
- ✅ Чаты отделов
- ✅ Push-уведомления
- ✅ Оплата через Click/Payme
- ✅ Mobile приложения в App Store и Google Play
- ✅ Web-приложение

**Бизнес:**
- ✅ 3-5 пилотных клиентов в работе
- ✅ Сайт и тарифы опубликованы
- ✅ Юридическая база (оферта, ПД, DPA)
- ✅ Базовая поддержка (email/чат)
